Culture Ontology: Description back to ToC

The Culture ontology (aka DRK ontology) is designed for the Culture Dataspace, providing extensible semantic concepts, properties, and their interrelationships for modeling metadata in the culture domain. It comprises common modules for cross-cutting concepts (such as organizations, places, creative works, events, and roles) and domain-specific modules (such as those tailored to theater showtimes). The ontology is extensible and allows the addition of new modules, for sectors such as museums or music marketplaces, as future use cases arise.

Culture IM's Theater Showtimes Model

The Culture IM's Theater Showtimes model comprises the Theater Showtimes module of the Culture Ontology and its associated general application profile (TS-GAP). Built on standards like Schema.org and DCAT, it defines structured concepts and relationships for theatrical productions and events.

Culture Ontology's Theater Showtimes Module

The Theater Showtimes module enhances the Culture ontology's core modules - Organization, Place, CreativeWork, Event, and Role - with specialized classes and properties for modeling theater-specific entities. Key classes include:

  • drk:TheaterOrganization: Represents a performing arts institution that creates or organizes theatrical productions and events.
  • drk:PlayProduction: Captures the entire lifecycle of a theatrical work, excluding the public performance.
  • drk:TheatricalEvent: Represents an actual theatrical performance event.
  • drk:PerformingArtsTheater: Models the theater venue itself.

The module reuses the Role module to articulate detailed concepts such as drk:Character, drk:Role, drk:CharacterPerformanceRole, and drk:ProductionRole, alongside properties like drk:characterRole, drk:creatorRole, and drk:eventRole to differentiate between onstage and offstage contributions.

Key Classes

drk:TheatricalProduction

This class encompasses the creation and development of a theatrical work, detailing production activities like writing, casting, and rehearsals, while excluding the final public performance. This class extends schema:CreativeWork and includes essential attributes such as:

  • Title
  • Description
  • Genre
  • Production type
  • Involved characters and roles
  • Languages
  • Audience type
  • Accessibility features
  • Sources of inspiration

This structured metadata aids cultural organizations in uniformly describing productions, enhancing discoverability and interoperability.

drk:TheatricalEvent

This class models the public presentation of a theatrical production, extending schema:TheatricalEvent, and links back to the corresponding drk:TheatricalProduction through the schema:workPerformed property. It captures event-specific details such as:

  • Venue
  • Dates (start and end)
  • Door opening times
  • Duration
  • Intermissions
  • Ticket offers
  • Languages spoken and subtitled
  • Character performances
  • Involved production roles

Standardized through controlled vocabularies and ISO formats, this class ensures a consistent, machine-readable representation of showtimes across theater organizations.
